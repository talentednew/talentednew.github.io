GitBasics

GitBasics

1.1 git init

1. For instance, if we want to create one folder with the name as git-demo, just type 'mkdir git-demo'
2. Go to the directory of target folder 'cd git-demo'
3. The third step is to input 'git init', this order will create one .git directory under git-demo
4. Give the input 'ls -la', this will show one .git directory which is one repository
5. These are the whole steps to initialize one git repository

1.2 git add

1. After the initialization of the git repository, we can add any files as desired
2. Under this case, if we add one file called index.html, we should input 'touch index.html'
3. For now, run 'git status -sb' will show ?? symbols in front of file, this renders that git does not know how to deal with those files
4. Use git add to add the files into one buffered area
5. There are two ways to finish the add process
    a. use 'git add index.html' to add all the files multiple times
    b. use 'git add .' to add all the files in one time
6. Running 'git status -sb' will show ?? transformed to A
7. At this point, git add process has been finished

1.3 git commit -v

1. Use 'git commit -m "commit message"' to commit a snapshot of all changes in the working directory
2. This is a shortcut command that immediately creates a commit with a passed commit message
3. Passing the -m option will forgo the text editor prompt in-favor of an inline message
